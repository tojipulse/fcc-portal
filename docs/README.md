# FCC PORTAL

> 少年サッカーチーム運営支援システム

---

# 1. プロジェクト概要

FCC PORTAL は、少年サッカーチーム運営に必要となる様々な業務を一元管理するWebシステムです。

現在LINE・Googleフォーム・Excel・紙・口頭で管理している情報を集約し、
保護者・コーチ・管理者が同じ情報をリアルタイムに共有できることを目的としています。

本システムは単なるスケジュール管理ではなく、

- 選手管理
- 家庭管理
- 出欠管理
- コーチ出席管理
- お知らせ
- 年度更新
- マスター管理
- 添付ファイル管理

までを含めた運営支援システムとして開発します。

---

# 2. 開発目的

本プロジェクトの目的は、

「運営負荷を最小限にしながら、
誰でも迷わず使えるシステム」

を実現することです。

特に以下を重視しています。

- スマートフォンだけで操作できる
- 保護者は迷わない
- コーチの運営負荷を減らす
- 管理者だけが高度な機能を利用できる
- 年度更新を数クリックで完了できる
- 将来の機能追加に耐えられるDB設計

---

# 3. システムコンセプト

FCC PORTAL は

**「保守性」「拡張性」「運用性」**

を最優先として設計しています。

開発途中だけではなく、

5年後・10年後も運用できることを前提に設計しています。

FCC PORTALでは以下のロールを採用します。

|ロール|概要|
|------|------|
|保護者|出欠回答・予定確認|

# 6. 開発方針

本プロジェクトでは

Repositoryパターン

を採用します。

画面から直接DBへアクセスしません。

---

# 7. システム構成

本システムは Next.js(App Router) を採用しています。

```text
Client
    │
    ▼
Page (UI)
    │
    ▼
Component
    │
    ▼
Repository
    │
    ▼
Database
```

UIから直接SQLを実行することは禁止しています。

Repositoryがデータアクセスを一元管理します。

# 11. 開発ルール

本プロジェクトでは以下を徹底します。

- Repository経由のみDBアクセス
- PageからSQL禁止
- ComponentからSQL禁止
- TypeScriptを使用
- 型安全を維持
- Feature単位で開発
- 共通Componentを積極利用
- 論理削除を採用
- 楽観ロック(version)を採用
- 監査項目(created_at等)を全テーブルへ追加

---

# 12. ChatGPT運用ルール

本プロジェクトはChatGPTとの共同開発を前提としています。

コード提供時のルール

- 全文コピー・全文貼り付け方式
- 差分修正は禁止
- 長い場合は前半・後半に分割
- ファイル単位で提供

設計変更時

- 先に設計書更新
- その後コード修正
